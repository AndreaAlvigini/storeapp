# ESERCITAZIONE FINALE

L'esercitazione finale riguarderà l'implementazione di un progetto già creato.

Ci saranno due gruppi, ogni gruppo avrà un **responsabile** i cui compiti saranno:

- 1 Responsabile **documentazione**, in quanto ogni fase ed ogni modifica od implementazione sarà dpocumentata seguendo lo schema del progetto principale

- 2 Responsabile della gestione dei **commenti nel codice**

- 3 Responsabile del **versionamento**, dovrà mettere a disposizione uno spazio Github condiviso ed accettare i partecipanti del proprio gruppo come collaboratori. Ciascun collaboratore utilizzarà i propri branch per le modifiche che verranno integrati nel branch Main attraverso fasi di Merge concordate

- 4 Il responsabile potrà delegare (quindi appaltando alcuni delle proprie responsabilità ad aòtri membri del team).
Inoltre, dato che ogni membro del Team avrà un compito specifico (es responsabile Front End), il responsabile (Capogruppo) potrà sotto delegare quindi invertire o scambiare le mansioni dei partecipanti del proprio gruppo.

# Formazione dei gruppi

1 - Due persone scelgono i partecipanti delle squadre (metodo giardinetti)

2 - Una volta che il gruppo è stato formato si decreta democraticamente un Capogruppo

3 - Il capogruppo assieme ai partecipanti decreta i vari task e le responsabilità di ogni collaboratore

# Task disponibili per ogni gruppo

1 - Capogruppo (coordinamento), ovviamente avrà l'obbligo di presenza o quantomeno dovrà coordinare il Team in caso di assenza (ad esempio il giorno in cui arriverà in ritardo dovrà premurarsi di delegare un membro del team a fare le sue veci o almeno dovrà dare indicazioni sulla svolgimento del lavoro in sua assenza)

## DA ADESSO IN POI IL LAVORO PROCEDERà AL RITMO IMPOSTO DAL CAPOGRUPPO

# PRIMA DI INCOMINCIARE

1 - Una volta che si ha l'idea chiara del progetto (ad avvenuta pianificazione) il gruppo dovra scegliere:

- Il nome dell'applicazione
- Area di intervento (applicazione store per collezionismo di figurine)
- Una palette di colori identificativa
- Creazione dello spazio github dedicato al progetto ed offerto dal capogruppo

Questo al fine di creare fin da subito una chiara identità del progetto

# REQUISITI DEL PROGETTO

- 1 Applicazione JAVA completa di front end seguendo lo schema presentato del cliente

1 - Dovranno essere implementate le tabelle del database (al momento sono "PRODOTTI" "CLIENTI" "ACQUII")

2 - Dovranno essere implementate le immagini per i prodotti (il gestore del sito dovrà poter aggiornare le immagini dei prodotti)

3 - Il front end dovrà essere creato tramite bootstrap (livello di qualità minimo esercitazione di ripasso di bootstrap portale web rosso)

4 - Backend: dovranno essere implementate le funzionalità di creazione e gestione dei prodotti o clienti etc.. in quanto i file iniziali possidono solamente le funzionalità di visualizzazione dati
